# ver.delta 상세 리드미

## 1) 버전 개요
- **핵심 목표**: 전통적 분류기로 감정 코드를 추정하고 4D 감정 벡터로 매핑.
- **중심 접근**: TF-IDF + LogisticRegression → 감정 코드(E10~) → EMOTION_VEC 매핑.

## 2) 핵심 파일 구성
- `ver.delta/4.1.1/main.ipynb`
  - `emovec_autorun_cls.py` 형태의 분류기 학습 파이프라인.
  - 모델 저장, 리포트 출력(accuracy, macro-F1, MAE) 포함.

## 3) 핵심 아이디어 정리
- LLM 대신 **경량 분류기 기반 감정 벡터 추출**을 시도.
- 감정 코드 → 4D 벡터 매핑으로 **SNN 입력 연결** 가능.

## 4) 이전 버전 대비 개선/특징
- LLM 비용을 줄이고 로컬에서 재현 가능한 **전통 ML 접근**을 강조.

## 5) 실행 방법 (처음 보는 사람용)
- **노트북 실행**
  1. `ver.delta/4.1.1/main.ipynb`를 열고 셀 순서대로 실행.
  2. 상단 출력 리포트에서 정확도/MAE 확인.

## 6) 한계 및 개선 아이디어
- 감정 코드 라벨 분포 불균형 가능 → **샘플링/가중치 개선** 필요.
- char_wb ngram 중심이어서 의미 기반 문맥 파악이 약함 → **임베딩 기반 분류기** 고려.

## 7) 다음 단계 제안
- 성능 지표(accuracy, MAE)를 기준으로 감정 벡터 매핑 반복 개선.
- 감정 코드 대신 직접 4D 회귀 모델 시도.
