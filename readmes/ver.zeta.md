# ver.zeta 상세 리드미

## 1) 버전 개요
- **핵심 목표**: 감정 분류 + SNN + 시각화를 통합해 하나의 실행 흐름으로 확인.
- **중심 접근**: 수동 입력 브릿지 + 통합형 시뮬레이터(감정 분류/도파민 쉴드/시각화).

## 2) 핵심 파일 구성
- `ver. zeta/main.py`
  - 외부 LLM이 제공한 4D 감정 벡터를 직접 입력 → SNN 처리 → 페르소나 프롬프트 생성.
- `ver. zeta/감정_잘됨.py`
  - 감정 분류(ML) → 4D 벡터 → SNN → 실시간 시각화까지 한 파일에 통합.
- `ver. zeta/감성대화말뭉치(최종데이터)_Training.json`
  - 분류기 학습용 감성 대화 데이터.

## 3) 핵심 아이디어 정리
- **Manual Bridge**: LLM과 SNN 사이를 사람이 직접 연결.
- **도파민 쉴드**: 억제성 뉴런의 과도한 평준화를 방지해 긍정 감정이 너무 약해지지 않도록 설계.
- **통합 실행**: 감정 분류 → SNN → 시각화를 한 번에 확인.

## 4) 이전 버전 대비 개선/특징
- eta의 드라마틱/수동 모드에서 **분류기+시각화를 통합**한 최신 데모 흐름.
- 감정 벡터 산출과 SNN 동작을 한 파일에서 확인할 수 있어 디버깅이 용이.

## 5) 실행 방법 (처음 보는 사람용)
- **Manual Bridge 실행**
  ```bash
  python "ver. zeta/main.py"
  ```
- **통합형(시각화 포함) 실행**
  ```bash
  python "ver. zeta/감정_잘됨.py"
  ```

## 6) 한계 및 개선 아이디어
- 통합형 파일의 규모가 커져 유지보수 어려움 → **모듈 분리** 필요.
- 특정 문장에 대한 강제 감정 주입 등 **테스트 로직 분리** 필요.
- ML/시각화 의존성이 많아 환경 설정 문서화 필요.

## 7) 다음 단계 제안
- 감정 분류 모듈과 SNN 모듈 분리 후 공통 인터페이스 정의.
- requirements 문서화 및 예제 실행 환경 스크립트 제공.
