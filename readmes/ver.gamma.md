# ver.gamma 상세 리드미

## 1) 버전 개요
- **핵심 목표**: SNN 구조의 프로토타입을 빠르게 설계하고 흐름(FLOW)을 검증.
- **중심 접근**: LLM → NTL(신경전달물질 코드) → K(막전위) → 뉴런 전파 흐름.

## 2) 핵심 파일 구성
- `ver.gamma/3.1.1/main.ipynb`
  - 초기 SNN 뼈대, 감정 저장소, 뉴런 클래스, 노드 분할, FLOW 예시.
- `ver.gamma/3.1.2/main.ipynb`
  - LLM 기반 JSON 파서 강화, 안전한 추출 로직 추가.
  - 동일한 SNN 뼈대 구조에 안정성 보완.

## 3) 핵심 아이디어 정리
- **감정 저장소(emotion_storage)**에서 NTL/K 누적과 감쇠를 관리.
- **뉴런 연결망**을 통해 감정 신호가 전달되며 가소성/연결 확장 개념을 탐색.
- 3.1.2에서 **LLM JSON 파싱 안전성 강화**.

## 4) 이전 버전 대비 개선/특징
- beta보다 낮은 구현 단계이지만 **SNN 구조를 명확히 형식화**하려는 버전.
- **LLM 출력 파싱 안정성** 개선은 이후 버전으로 넘어가는 중요한 기반.

## 5) 실행 방법 (처음 보는 사람용)
- **노트북 실행**
  1. `ver.gamma/3.1.1/main.ipynb` 또는 `ver.gamma/3.1.2/main.ipynb` 실행.
  2. FLOW 섹션에서 입력 → 감정 저장소 → 뉴런 자극 흐름 확인.

## 6) 한계 및 개선 아이디어
- `do_work` 등 **미완성 메서드**가 존재해 실제 계산 로직 필요.
- NTL/K 업데이트 수식이 불안정할 수 있어 **클리핑/스케일링** 필요.
- LLM 의존성이 크므로 **대체 분류기/규칙 모델** 준비 필요.

## 7) 다음 단계 제안
- 뉴런 고유 연산(do_work) 정의 및 에너지/감쇠 규칙 실험.
- 감정 추출부를 분류기 기반으로 대체하는 경량화 실험.
