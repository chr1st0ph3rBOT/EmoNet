# ver.eta 상세 리드미

## 1) 버전 개요
- **핵심 목표**: 감정 변화를 크게 체감시키는 드라마틱 모드 + 시각화/로그 기능 강화.
- **중심 접근**: 수동 감정 입력 + 고밀도 SNN + 감정 궤적 시각화/CSV 로그 저장.

## 2) 핵심 파일 구성
- `ver.eta/main_뒤끝있는 버전.py`
  - 드라마틱 감정 변화 버전(강한 감정 증폭, 직설적 프롬프트 지시 포함).
  - 시각화 및 로그 저장 포함.
- `ver.eta/main_기억 없는 버전_성공!.py`
  - 키워드 기반 감정 추출 + SNN 처리 + 프롬프트 생성.
- `ver.eta/시각화 추가 버전.py`
  - 루프 안정화, refractory period/energy decay 적용, 자동 감정 예측 결합.
- `ver.eta/asd.py`
  - 학습된 모델(.pkl) 상태 점검 스크립트.

## 3) 핵심 아이디어 정리
- **감정 증폭 전략**: 입력/뇌 출력의 가중치 조합을 강하게 적용해 감정 변화를 크게 만듦.
- **시각화/로그 기반 분석**: 감정 변화와 네트워크 활성 흐름을 그래프로 추적.
- **수동 LLM 입력**: 외부 LLM 점수를 사람이 직접 주입하는 브릿지 방식.

## 4) 이전 버전 대비 개선/특징
- epsilon 대비 **체감형 데모 기능(시각화/로그)** 강화.
- 감정 표현 톤을 직접적으로 변경하는 지시문을 강화.

## 5) 실행 방법 (처음 보는 사람용)
- **드라마틱 모드 실행**
  ```bash
  python "ver.eta/main_뒤끝있는 버전.py"
  ```
- **기억 없는 버전 실행**
  ```bash
  python "ver.eta/main_기억 없는 버전_성공!.py"
  ```
- **모델 상태 점검**
  ```bash
  python ver.eta/asd.py
  ```

## 6) 한계 및 개선 아이디어
- 감정 증폭이 과도할 수 있어 **강도 조절 파라미터** 필요.
- 수동 감정 입력은 일관성 부족 → 자동 분류기 결합 필요.
- 시각화/시뮬레이션 로직 분리로 유지보수성 개선 필요.

## 7) 다음 단계 제안
- 드라마틱 모드와 일반 모드의 스위치 분리.
- 시각화 모듈을 독립 패키지로 분리하여 재사용성 확보.
