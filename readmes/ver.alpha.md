# ver.alpha 상세 리드미

## 1) 버전 개요
- **핵심 목표**: 텍스트를 바로 신경전달물질 스파이크(λ 기반)로 변환하는 실험적 파이프라인 구축.
- **중심 접근**: 임베딩 유사도 → 뉴로모듈레이터 점수 → 발화율(λ) → 포아송 스파이크 생성.

## 2) 핵심 파일 구성
- `ver.alpha/ver.1/single_file_emotion_spike.py`
  - 임베딩 기반 점수 산출, λ 계산, 포아송 스파이크, RL 업데이트, 요약 생성까지 한 파일에 통합.
- `ver.alpha/emotion_dataset.txt`
  - 발화 텍스트 ↔ 이모지/기호 반응 쌍 데이터.
- `ver.alpha/emotion_dataset_contextual.txt`
  - **이전 문맥 2개 + 현재 발화 + 반응** 형태의 컨텍스트 데이터.
- `ver.alpha/ver.2/*.txt`
  - 카카오톡 대화 로그 원문(실제 대화 흐름 참고용).

## 3) 핵심 아이디어 정리
- **스파이크 기반 감정 표현**: 감정을 연속값 벡터가 아닌 “스파이크 활동”으로 표현한다는 가정.
- **임베딩-시드 매핑**: 뉴로모듈레이터별 의미 시드 문장을 임베딩하여 유사도 기반 점수 산출.
- **요약 계층**: 스파이크/점수를 다시 자연어 요약으로 변환하여 해석 가능성을 확보.

## 4) 이전 버전 대비 개선/특징
- 초기 단계이지만 **스파이크 생성 → 요약**까지 한 번에 연결한 파이프라인이 존재.
- **RL 업데이트(α/β)**로 감정 반응의 학습 가능성을 실험.

## 5) 실행 방법 (처음 보는 사람용)
- **스파이크 파이프라인 데모 실행**
  ```bash
  python ver.alpha/ver.1/single_file_emotion_spike.py
  ```
- **데이터 확인**
  - `ver.alpha/emotion_dataset.txt`
  - `ver.alpha/emotion_dataset_contextual.txt`

## 6) 한계 및 개선 아이디어
- **시드 문구 의존성**이 크므로 시드 다양화/튜닝 필요.
- **λ 스케일 안정성**(alpha/beta 범위) 관리가 필요.
- 스파이크를 최종 감정 벡터/응답 톤으로 연결하는 **해석 레이어**가 부족.

## 7) 다음 단계 제안
- 시드/임베딩 개선 + 스파이크 결과를 4D 감정 벡터로 환산하는 모듈 추가.
- alpha/beta 안정화(클리핑, 스케줄링, 정규화) 전략 도입.
